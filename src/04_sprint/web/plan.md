# 语料库检索系统 Web 界面实施计划

## 设计方案与技术选型
本计划旨在构建一个结构清晰、界面友好且功能完备的语料库搜索系统。我们将采用前后端分离的现代化架构：

1. **前端 (Front-end)**: 
   使用原生 HTML, CSS, JavaScript 构建。界面设计注重简洁性和学术专业感，核心为一个居中的大尺寸搜索框，下方配备显眼的“仅搜索已标注数据 (Annotated-only)” 复选框，满足随时切换搜索范围的核心需求。
   请求将通过 `fetch` API 或 `axios` 发送表单信息至后端。

2. **后端 (Back-end)**: 
   采用 Python 的 **FastAPI** 框架。它具有高性能、易于编写和自动生成 API 文档等优点。后端将负责提供 `/search` 等 RESTful 接口，接收前端传来的 Query 和 `is_annotated` 请求参数，并对接底层检索引擎。

3. **检索引擎与执行**: 
   拟使用 **Whoosh** (或 Elasticsearch) 对原始语料和标注数据进行全文索引。针对“仅搜索标注数据”功能，我们会在建立索引时为每段文本添加一个布尔类型的 `is_annotated` 字段。
   当前端传回包含该过滤器选项的搜索请求时，后端会将条件组装查询并传递给 Whoosh，从而直接返回精确结果（文本段落、匹配高亮以及相关统计数值），并在需要时提供用于渲染可选数据图表的 JSON 响应。
